Adds a conditional rule

addConditionalRule(mixed $class, string | array $main_columns, mixed $conditional_values, string | array $conditional_columns) : void

If a non-empty value is found in one of the `$main_columns`, or if specified, a value from the `$conditional_values` array, all of the `$conditional_columns` will also be required to have a value.

Parameters

$class

mixed

The class name or instance of the class this rule applies to

$main_columns

stringarray

The column(s) to check for a value

$conditional_values

mixed

If NULL, any value in the main column will trigger the conditional column(s), otherwise the value must match this scalar value or be present in the array of values

$conditional_columns

stringarray

The column(s) that are to be required

addRegexReplacement(mixed $class, string $search, string $replace) : void

Regex replacement is done after the `post::validate()` hook, and right before the messages are reordered. If a message is an empty string after replacement, it will be removed from the list of messages.

Parameters

$class

mixed

The class name or instance of the class the columns exists in

$search

string

The PCRE regex to search for - see http://php.net/pcre for details

$replace

string

The string to replace with - all $ and \ are used in back references and must be escaped with a \ when meant literally

Requires that a column have a non-`NULL` value

addRequiredRule(mixed $class, array $columns) : void

Before using this method, try setting the database column to `NOT NULL` and remove any default value. Such a configuration will trigger the same functionality as this method, and will enforce the rule on the database level for any other code that queries it.

Parameters

$class

mixed

The class name or instance of the class the column(s) exists in

$columns

array

The column or columns to check - each column will require a value

Restricts a column to having only a value from the list of valid values

addValidValuesRule(mixed $class, string $column, array $valid_values) : void

Please note that `NULL` values are always allowed, even if not listed in the `$valid_values` array, if the column is not set as `NOT NULL`. This functionality can also be accomplished by added a `CHECK` constraint on the column in the database, or using a MySQL `ENUM` data type.

Parameters

$class

mixed

The class name or instance of the class this rule applies to

$column

string

The column to validate

$valid_values

array

The valid values to check - NULL values are always allows if the column is not set to NOT NULL
